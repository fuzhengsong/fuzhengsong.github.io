发表于 2019-02-16 | 分类于 部署

可能会用到的命令：

  1. vi 修改文件
  2. wq 保存并退出 q退出 q！强制退出 wq！保存强制退出
  3. nginx -s reload 重启nginx
  4. nginx -s stop 停止nginx
  5. ps -ef|grep nginx 查看nginx进程
  6. curl 本地访问ip
  7. wget 下载资源
  8. netstat -ntpl 查看处于监听的端口
  9. netstat -anp 查看所有端口
  10. mv 重命名
  11. ln -s 软连接 xxx/xx/A /usr/bin/A 把A命令添加到全局启动

购买阿里云服务器

看个人实力决定。阿里有学生计划，如果可以学生认证就非常便宜了，只要9.9。

远程连接

购买了ECS服务之后，会生成一个实例。

点击远程连接，会给你一个6位数的密码，这个密码很重要，需要记住。后面连接服务器会用到。

这里我会再重置一下实例密码，这个在后面连接实例的时候同样需要验证。不知道默认的密码是什么，就重置一个自己能记住的密码吧。可以在更多>密码/密钥中有重置实例密码。

连接实例： 用户名是root 密码就是自己设置的

Termius 和 FileZilla

如果每次要连接服务器都要进阿里云的后台也是非常麻烦的，我们希望在自己的本地也能连接。这时候需要下载2个软件，Termius可以连接服务器实例，windows中可以用xshell。FileZilla可以帮我们上传文件到服务器，windows中可以用和xshell配套的xftp。使用上来说windows对这块支持比较好。

下载好Termius我们就可以连接服务器了, 点击new host，出现下面的登陆界面

label随便填，地址填自己阿里云服务公网ip， port都是默认的22， username 是root，密码是实例的密码（自己重置的那个），save完之后connect就可以连接上了。

下载FileZilla同理账号密码也是同样的。连接完成之后就能把本地的文件上传给服务器了

安装node、mongodb、nginx

执行以下命令：

wget https://nodejs.org/dist/v8.15.0/node-v8.15.0-linux-x64.tar.xz

tar xvf node-v8.15.0-linux-x64.tar.xz

mv node-v8.15.0-linux-x64 node

mongodb的下载时间会比较长，如果失败重新下载

wget https://fastdl.mongodb.org/linux/mongodb-linux-x86_64-3.4.5.tgz

tar zxvf mongodb-linux-x86_64-3.4.5.tgz

mv mongodb-linux-x86_64-3.4.5 mongodb

新建存放数据的文件

mkdir /root/mymongodb、mkdir /root/mymongodb/data

到这里node和mongodb就下载好了，先添加个软连接到全局，再继续安装nginx

ln -s /root/node/bin/node /usr/bin/node
ln -s /root/node/bin/npm /usr/bin/npm
ln -s /root/mongodb/bin/mongod /usr/bin/mongod

添加了软连接之后使用 node -v npm -v 查看版本

安装nginx

yum -y install make zlib zlib-devel gcc-c++ libtool openssl openssl-devel

cd /usr/local/src/

首先要安装 PCRE
wget http://downloads.sourceforge.net/project/pcre/pcre/8.35/pcre-8.35.tar.gz

tar zxvf pcre-8.35.tar.gz

./configure

make && make install

pcre-config –version

安装nginx

cd /usr/local/src/

wget http://nginx.org/download/nginx-1.6.2.tar.gz

tar zxvf nginx-1.6.2.tar.gz

cd nginx-1.6.2

./configure –prefix=/usr/local/webserver/nginx –with-http_stub_status_module –with-http_ssl_module –with-pcre=/usr/local/src/pcre-8.35

make

make install

ln -s /usr/local/webserver/nginx/sbin/nginx /usr/bin/nginx

查看nginx版本

nginx -v

配置nginx

进入配置文件
cd /usr/local/webserver/nginx/sbin/nginx/conf

vi nginx.conf

添加安全组

这个坑很大，之前配完mongodb启动，node服务启动，服务器本地curl成功，但是外网就是无法访问。最后查到是阿里云默认没有放行80端口，如果需要直接访问api接口，还需要放行后端接口，我这里是4000

具体配置如下：

上传文件

使用之前下载好的FileZilla，上传后端文件和前端打包好的文件，注意前端打包项目文件的地址需要和nginx中的目录对应，上传完成之后不要忘记npm install

启动项目

做完前面的工作，现在可以开始启动项目了。

启动mongodb

dbpath是存放数据的地址

mongod -dbpath=/root/mymongodb/data

执行该命令后出现等待连接 27017说明成功了

但是这样启动关闭和服务器的连接，mongodb的连接同样也会终止，我们需要一个常驻的进程，执行命令：

mongod -dbpath=/root/mymongodb/data -fork -logpath=/root/mymongodb/log/mongodb.log

这里需要一个logpath，存储日志

启动node服务

和mongodb一样，node也是需要常驻的，不然关了Termius页面就挂了
安装一个pm2来管理进程即可

pm2命令：

  1. pm2 start
  2. pm2 stop
  3. pm2 restart

npm i pm2 -g

进入存放项目的地址

cd /root/blog

pm2 start bin/www

这时候出现mongodb connect success 以及 linsten on 4000说明服务启动成功了
